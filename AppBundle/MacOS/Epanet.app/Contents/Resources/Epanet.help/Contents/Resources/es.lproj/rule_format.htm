Formato de las Leyes de Control basadas en Reglas

 Inicio  Anterior  Siguiente

Las Leyes de Control basadas en Reglas consisten en una serie de instrucciones del tipo:

RULE IDregla

IF  condición_1

AND condición_2

OR  condición_3

AND condición_4

etc.

THEN acción_1

AND  acción_2

etc.

ELSE acción_3

AND  acción_4

etc.

PRIORITY Prioridad

donde:

IDregla

 =

un identificativo ID asignado a la regla

condición_n

 =

una cláusula de condición

acción_n

 =

una cláusula de acción

Prioridad

 =

un orden de prioridad (p.ej. un número del 1 al 5)

 

Observaciones:

1.Solo las instrucciones RULE, IF y THEN de una ley de control son estrictamente requeridas; las otras partes son opcionales.
2.En una regla se pueden utilizar y combinar las cláusulas AND y OR tantas veces como se quiera.
3.Cuando se mezclan las cláusulas AND y OR, el operador OR tiene precedencia sobre el operador AND. Por ejemplo,

IF A or B and C

es equivalente a

IF (A or B) and C.

Si en realidad lo que se quería decir es IF A or (B and C)habrá que utilizar para ello las dos reglas siguientes:

Regla 1: IF A THEN ...

Regla 2: IF B and C THEN ...

4.El orden indicado en PRIORITY se utiliza para determinar qué reglas son aplicables cuando dos o más acciones sobre una línea entran en conflicto. Una regla que no tenga una prioridad especificada siempre tendrá menor prioridad que otra que sí que la tenga. Para dos reglas con la misma prioridad, prima la que aparece en primer lugar dentro de la sección.
5.No hay límite en cuanto al número de leyes de control que se pueden añadir a un proyecto.
